Esta API fue disenada para soportar multiples clientes con distintas necesidades de permisos.

Se implementaron middleware de autenticacion, auditoria de eventos y politicas de autorizacion por recurso. Adicionalmente, se incluyeron pruebas de integracion para endpoints criticos.

La base de codigo permite escalar nuevos modulos sin duplicar logica de seguridad o validacion.
